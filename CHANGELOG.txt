# Changelog

SCC v0.8.6.68 - WEC 2014 Silverstone real import (LMP1-H/L split confirmed live)

- Imported wec-silverstone-2014 (round 1) for real: 27 entries with correct overall pos 1-27,
  confirmed LMP1-H/LMP1-L class split appearing in both the race classification and
  qualifying documents (Toyota #8 TS040 Hybrid's debut win), 27-row qualifying, 13 pole
  records, 69-row FL Classification.
- js/importer.js qualifyingAverageDriversFromText: some seasons (2014 confirmed) publish
  qualifying pole-driver data as a "FastestLapByDriverAfter...ByClass" document ranking each
  driver's single best lap of the WHOLE weekend so far, not just the qualifying session -
  each row has its own trailing Session column ("Qualifying Practice" vs "Free Practice 3"
  etc). Without filtering to that column, a car's unused 3rd crew member's practice-session
  personal best could wrongly count as one of the two actual qualifying nominees. Now filters
  out Free Practice/Warm-Up rows before attributing pole drivers - improved 2014 Silverstone's
  precise 2-driver pole attribution from 0/27 cars to 21/27 (remaining 6 cars have no
  qualifying-session lap in this ranking at all - their actual 2nd nominee's official lap
  wasn't their personal best of the weekend - falls back to crediting the full listed crew).

SCC v0.8.6.67 - WEC 2013 Silverstone real import (two-driver-average qualifying)

- Imported wec-silverstone-2013 (round 1) for real, replacing the skeleton entry: 31 entries
  (correct combined-classes overall pos 1-31, computed the same way as Sebring 2012), 30-row
  qualifying order, 88-row FL Classification, 10 pole records across 4 classes + Overall.
- 2013 was the first WEC season with two-driver-average qualifying (grid set by averaging
  each car's two nominated drivers' best laps, not a single fastest lap) - the parser
  functions for this (qualifyingAverageRowsFromText/qualifyingAverageDriversFromText/
  polesFromAverageRows) already existed from earlier work on the 2018-2019 era but had never
  actually been wired into a real import. First real use here: needed a SEPARATE qualifying-
  session Fastest-Lap-by-Driver document (not the race one - a different Al Kamel file,
  "07_FastestLapByDriver_Qualifying Practice.PDF") to correctly attribute each pole to the
  two drivers actually nominated for qualifying, since the Drivers cell in the qualifying
  classification itself lists the whole crew (2 or 3 names) undifferentiated.

SCC v0.8.6.66 - WEC 2012-2017 era parser support + first real historical import (Sebring 2012)

- Built an offline Node-based PDF import pipeline (download via curl, extract text with
  pdfjs-dist matching js/app.js's readPdfText exactly, run the real unmodified parsing
  functions via the existing __SCC_IMPORTER_INTERNAL__ test hook) to verify historical
  imports without a live browser - reusable for the remaining WEC/ELMS/MLMC/IMSA seasons.
- js/importer.js fixes, all found and verified against real 2012 WEC Sebring PDFs:
  - Class names printed with a space ("LM P1", "LM GTE Pro") now recognised, not just the
    modern no-space form ("LMP1", "GTEPRO").
  - Real bug in parseClassification: once a document's "Not classified" section flipped an
    internal status flag, later classified rows in a DIFFERENT class group were never
    correctly recognised again (their own Pos column got misread as the car number). Fixed
    by detecting a classified row from its own column shape instead of a flag that never
    reset. Also added "Excluded" as a recognised status-section header (stewards
    disqualification), alongside the existing Not classified/Retired/Disqualified.
  - Driver names in "Fastest Lap by Driver" documents from this era print as an ordinary
    "Firstname Surname" cell (neither initials nor all-caps), a shape none of the existing
    name regexes covered and unsafe to free-text-scan for (a model fragment like "Ferrari
    F458 Italia LM GTE Pro" could accidentally look like one). Added driverCellAfterClass(),
    which locates the Driver cell by its structural tab-column position (right after Class)
    instead of guessing from free text - wired into every function that reads driver names
    from this document type (timeRecordsFromText, qualifyingTimeDriverMap,
    flClassificationFromText, fullNameEnrichmentMap).
  - Lap/segment times in this era use an apostrophe for the final minute-to-second separator
    ("1'47.534", "12:00'15.243") instead of the modern all-colon format - every time-matching
    regex across parseClassification/qualifyingRowsFromText/timeRecordsFromText/
    qualifyingTimeDriverMap/flClassificationFromText/parseStartingGrid now accepts both and
    normalises back to colon form before storing (toColonTime()), so nothing downstream needs
    to know the source format.
- js/app.js normClass: added explicit LMP2/LMP3/LMP1H/LMP1L branches (LMP2 in particular had
  no branch at all and silently fell through un-normalised for any input with spaces/hyphens
  - a latent bug independent of this era's work, caught by a round-trip check against every
    SERIES class id).
- Imported wec-sebring-2012 (60th Annual 12 Hours of Sebring, WEC's first-ever season opener)
  for real, replacing the skeleton entry: 29 classified/NC/DSQ entries, full qualifying order
  (28 cars), 82-row driver-level FL Classification, poles and fastest laps per class - all
  cross-checked by hand against the source PDFs.
- Fixed after user spotted it live in the Order tab: entries[].pos must be the genuine
  overall race position (unique 1..N across every class), not the per-class rank the source
  "ClassificationByClass" PDF prints in its own leading column - storing the raw per-class
  rank made every class's own P1 car tie at overall pos 1 (four "WINNER OVERALL" labels
  instead of one, wrong Pos column). Recomputed by sorting all entries together (status, then
  laps descending, then total time ascending, then car number) and re-verified directly
  against orderRowsHtml() - exactly one "Winner overall" label, correct 1-2-3-4-5-6 Pos
  column. Only entries[].pos needed this - qualifying pos is intentionally per-class already
  (no combined-classes qualifying position exists in real timing).

SCC v0.8.6.58 - IMSA WeatherTech SportsCar Championship support (new series)

- New importer support for imsa.results.alkamelcloud.com, a genuinely
  different results vendor product from the *.alkamelsystems.com sites
  (WEC/ELMS/MLMC) with its own document shape end to end:
  - ONE combined all-classes ranking per session (like ELMS' 90_ doc), but
    column order [Pos, Pic, Nr, Class, Drivers, Team, Car, ...] with Drivers
    BEFORE Team (opposite of every Al Kamel doc) and no separate Car/model
    column in the qualifying doc at all.
  - A real print-width line-wrapping problem the other vendor doesn't have -
    a car's Drivers and/or Car text that doesn't fit the column spills onto
    a following physical line with no Pos/Pic/Nr/Class prefix of its own,
    sometimes landing AFTER the row's own trailing numeric columns once
    reassembled. New imsaMergeWrappedLines()/imsaRowFields() reconstruct one
    clean logical line per car first (driver names pulled out by content
    from the whole line, not just the tail) so every other extraction step
    works the same way it does for every other document in this file.
  - GTD/GTD PRO rows carrying a Bronze-rated driver print an extra
    single-letter "B" marker cell right after Class (same shape as WEC's
    Hybrid "H"/Pro-Am "P/A" marker cells) - skipped, not needed here.
  - A driver name format neither existing regex handled: Title-Case
    surnames with a lower-case connector ("R. van der Zande", "A. Felix da
    Costa"), sometimes mixed with upper-case connectors ("P. Di Resta") in
    the same document. New case-insensitive IMSA_DRIVER_NAME_RE, with a
    narrow trailing-connector extension (only "da"/"de", not "van der")
    specifically to avoid corrupting the one real team whose name literally
    repeats a driver's own connector-surname ("van der Steur Racing" /
    driver R. van der Steur).
  - Gap/Gap-to-previous columns can be a lap count, a M:SS.mmm time once
    the gap passes a minute, a bare "-" for the class leader, or (for a car
    that stopped running late but stayed classified) an extra "Not Running"
    status word inserted as its own column.
  - New imsaFlByDriverRowsFromText() for the separate "Results by Driver
    Fastest Lap" document - a much simpler layout (no wrapping at all) but
    fuses the car number and team into one cell.
- Added SERIES.imsa's LMP2 class (was missing entirely - IMSA runs GTP,
  LMP2, GTD PRO, GTD) and fixed its GTD PRO class key (GTD_PRO -> GTDPRO,
  matching what the class-detection patterns actually normalise to -
  GLOBAL_CLASS_META already had both spellings as a fallback, which is
  what masked this until LMP2 rows made the real gap visible).
- Verified end to end on two real, freshly-fetched rounds: Rolex 24 at
  Daytona 2026 (R1, 24-hour race with hourly result subfolders - 60/60
  entries, 60/60 qualifying rows, all 4 classes, clean team/model text and
  constructor logos throughout) and WeatherTech Raceway Laguna Seca 2026
  (R4, sprint race, no LMP2 this round - 33/33 entries and qualifying).
  No console errors on either import.

SCC v0.8.6.56 - FL Classification: merged GTE Pro/Am and LMP2/Pro-Am sections

  "X Fastest Laps" sections, each strictly sorted within itself - so a

  within the merged list. Every other class is untouched. The per-class
  "Average laptime" cards above the list are unaffected - still computed
  and shown separately per real sub-class, same as before.
- Verified on wec-spa-2018 (GTE Fastest Laps: Porsche/Ford/AF Corse Pro
  cars first, Dempsey-Proton's Am car correctly interleaved in by its own
  pace, not appended after all Pro cars) and elms-barcelona-2026 (LMP2 +
  LMP2PA merged into one "LMP2 Fastest Laps", LMP3/LMGT3 untouched). No

SCC v0.8.6.55 - Two-driver-average qualifying: team/model text + single-lap cars

- qualifyingAverageRowsFromText (2013-2020 era) built team/model/drivers by
  whitespace-tokenising the whole line and relying on modelFromLine's
  hardcoded Hypercar-era model list to find where the model name ended -
  that list has no idea about GTE-era cars ("Ford GT", "BMW M8 GTE",
  "Porsche 911 RSR"...), so for any car whose model it didn't recognise,
  team/model/class/times all ran together into one field ("BMW Team MTEK
  BMW M8 GTE LMGTE Pro 2:13.683 2:14.351 2 2'14.017" shown as the team
  name). Same document already prints Team/Drivers/Car/Class in their own
  tab cells (same layout qualifyingRowsFromText and parseClassification
  already use) - switched to reading those cells directly, keeping the
  numeric-sequence regex only for the trailing driver1/driver2/laps/average
  columns (still needed to survive the Hybrid era's extra "H" marker cell).
- A car whose 2nd nominated driver had every lap deleted (track limits,
  steward decision) has only ONE recorded time, not two - the sequence
  regex required exactly 4 numeric values and silently skipped these cars
  entirely, dropping a real recorded qualifying time and pushing them into
  the "beyond recorded time" grid-order fallback instead (e.g. Gulf Racing
  UK #86 at Spa 2018-19, genuinely 10th on qualifying pace, was showing as
  20th/unclassified). Added a 3-value fallback pattern (one time, laps,
  average - which just repeats that one time) for this case.
- Re-imported wec-spa-2018 and wec-silverstone-2019 (both already using
  this parser) with the fix: team/model text now clean throughout both
  (verified GTE Pro/Am and LMP1/LMP2 rows), car #86 correctly placed at

SCC v0.8.6.41 - Importer: series without a Hyperpole shootout (ELMS, MLMC, pre-2024 WEC)

- The qualifying importer only ever worked for WEC's Hyperpole format. Series that
  instead run one plain qualifying session per class (ELMS, MLMC) or WEC's own
  2022-2023 format (three 15-minute class sessions, no shootout at all) produced
  wrong or missing qualifying rows - reported by the user importing ELMS 2026
  Round 1 Barcelona, whose race results imported fine but qualifying didn't.
  No format-specific code was needed: every one of these is structurally the same
  "one Classification + one Fastest-Lap-by-Driver PDF per class" shape the
  Hyperpole path already understood, so the same qualifying importer now just
  handles it correctly instead of needing a separate path. Root causes fixed:
  - classFromText() picked whichever class pattern matched EARLIEST in a row's
    text, so a model name containing a shorter class name as a bare substring
    ("Porsche 911 GT3 R LMGT3" has "GT3" before its own "LMGT3") could win over
    the row's real class column further right. Now takes the rightmost
    non-contained match instead - every AlKamel row prints its class column
    after the model, so the real class always wins.
  - classifyDoc() scanned a document's ENTIRE text for "fastest lap", which
    every Classification doc's own footer annotation ("Fastest Lap Lap 8")
    contains too, misclassifying plain Classification PDFs as Fastest-Lap-by-
    Driver ones. Now checks the filename first (Al Kamel's own naming is
    consistent and far more reliable) and only falls back to a narrow
    title/header text scan.
  - qualifyingRowsFromText() re-tokenised the whitespace-collapsed row instead
    of using the same tab-isolated Drivers/Car/Cat columns the Race
    Classification parser already trusts, so a two-word surname with no
    recognised connector ("Valentin HASSE CLOT") lost its second word whenever
    more row text followed the driver list (every row in a non-Hyperpole doc,
    since there's no Hyperpole-only "one name, then nothing else" shape here).
  - singleClassFromText() and classFromText() both mistook a more specific
    class match ("LMP2 Pro/Am", "LMGTE Am") for two different classes when a
    shorter, more generic pattern ("LMP2", "Am") also matched inside the same
    text, wrongly declaring an unambiguous single-class document ambiguous, or
    picking the bare qualifier over the real class name.
  - Added LMGTE PRO / LMGTE AM class patterns (missing entirely - WEC dropped
    them after 2023) and a new GTEAM/GTEPRO class id, with LMGTE Am rendering
    in orange in WEC's class list.
  - normClass() was force-normalising any bare "GT3" to "LMGT3", which is
    correct for WEC/ELMS but wrong for MLMC, whose own Al Kamel documents
    print plain "GT3" as a genuinely different, already-existing class id.
  - Driver full-name extraction ("Ahmad AL HARTHY") dropped the second half of
    a two-word all-caps surname when its connector word ("AL") wasn't in the
    recognised list; added it alongside the existing initials-format fix.
- The "driver who set this time" bold/italic highlight (added earlier for
  Hyperpole) needed no change at all for non-Hyperpole series - it already
  works from the same Classification + Fastest-Lap-by-Driver PDF pair per
  class, and non-Hyperpole formats have no cutoff, so it applies to literally
  every car, not just a top-10 shootout.
- Verified against real official Al Kamel PDFs (not synthetic data) for ELMS
  2026 Round 1 Barcelona (4 classes), MLMC 2026 Round 1 Barcelona (2 classes),
  and WEC 2023 Round 1 Sebring (3 pre-Hyperpole class sessions, LMGTE Am
  included) - every class, car and qualifying time-setter came out correct,
  cross-checked against the Race Classification's own entry/class counts.
