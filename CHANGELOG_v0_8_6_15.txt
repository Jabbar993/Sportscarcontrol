SCC v0.8.6.15 · ACO importer fix

Actual changes in this ZIP:
- Added js/patch-v08615-aco-importer-fix.js and loaded it after previous importer patches.
- Static title/version updated to v0.8.6.15.
- Import wizard relabelled: no Entry List; Race Classification is the main results source; Hyperpole/Qualifying is the PP source.
- Results field accepts Race Classification / Results PDF and imports race order.
- Fastest laps are extracted from Race Classification when possible, so the FL PDF does not need to be uploaded separately.
- Qualifying/PP field accepts multiple PDFs at once for Hyperpole HYP + Hyperpole LMGT3.
- Import session status now confirms detected Results cars / FL rows / PP rows.
- Start new import / clear loaded PDFs resets session arrays, textareas, file inputs and previews.
- Changing Race ID clears stale importer session data.
- Save to database writes the current race into RACES and browser localStorage under sccSavedRaces.v0.8.6, then refreshes tree/selectors.
- Saved browser races are merged back into RACES on load.

Known limitation:
- Browser localStorage persistence is local to the browser. To make a permanent source-code/database export, use Export current race JSON after saving.
