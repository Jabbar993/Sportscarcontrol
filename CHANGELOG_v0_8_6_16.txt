SCC v0.8.6.16 - Imola duplicate/Juncadella importer cleanup

Verified changes in this ZIP:
- Version marker changed to v0.8.6.16 in index.html.
- database/drivers.json: removed duplicate driver record `daniel-juncadella`.
- database/drivers.json: `Dani Juncadella` keeps DOB 1991-05-07 and has aliases `Daniel Juncadella`, `D. Juncadella`.
- ACO importer normalizes `Daniel Juncadella` and `D. Juncadella` to `Dani Juncadella`.
- ACO result dedupe now merges race rows by car number first, not `car number + class`, because class can be missing/variant in PDF extraction and that was a likely source of duplicate Imola rows.
- When duplicate rows for one car are detected, driver lists are merged by canonical driver name instead of blindly keeping one crew.

Known limitation:
- This fixes the duplicate key/canonical-name cause found in the shipped ZIP. It does not claim external PDF validation; if a PDF row is still skipped (e.g. missing P4), that needs the exact PDF text extraction sample.
